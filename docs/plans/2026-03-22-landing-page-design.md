# 랜딩페이지 디자인 문서

## 개요

Admin Random Box 서비스를 소개하는 풀페이지 랜딩페이지를 루트 경로(`/`)에 추가한다.
비로그인 사용자에게는 랜딩페이지를, 로그인 사용자에게는 기존 앱 화면을 표시한다.

## 접근 방식

**방식 A: 루트 페이지 분기** 선택.

- 현재 `(user)` 그룹의 루트 `page.tsx`에서 인증 상태 확인
- 비로그인 → `<LandingPage />` 컴포넌트 렌더링
- 로그인 → 기존 캡슐 목록 렌더링
- 기존 구조 최소 변경, URL 깔끔(`/`), SEO 유리

## 섹션 구성

### 1. Hero
- 풀블리드 Unsplash 배경 이미지 + 다크 오버레이
- 핵심 카피: 랜덤박스의 짜릿함 전달
- CTA 버튼: "지금 시작하기" → `/signup`

### 2. 서비스 소개 (3단계)
- 구매 → 오픈 → 수령/분해
- 각 단계 아이콘 + 간결한 설명
- 스크롤 시 fade-in 애니메이션

### 3. 캡슐 쇼케이스
- 실제 캡슐 mock 데이터 활용
- 가격대, 등급별 상품 예시 표시
- 카드 그리드 레이아웃

### 4. 피버 게이지 소개
- 커뮤니티 피버 시스템 설명
- 현재 진행률 시각화 (게이지 바)
- 보상 상품 이미지

### 5. 명예의 전당
- S급 상품 당첨 후기 하이라이트
- 카드 레이아웃, 상품 이미지 + 리뷰 텍스트

### 6. CTA + Footer
- 회원가입/로그인 유도 CTA
- 서비스 정보, 링크

## 기술 스택

- **컴포넌트**: `src/components/user/LandingPage.tsx` (클라이언트 컴포넌트)
- **스타일링**: Tailwind CSS (프로젝트 기존 설정)
- **애니메이션**: Framer Motion (프로젝트에 이미 설치됨)
- **이미지**: Unsplash (next.config.ts에 이미 허용)
- **데이터**: 기존 mock 데이터 활용 (캡슐, 피버, 리뷰)
- **인증**: Zustand auth store 활용

## 스타일 방향

- 다크 톤 기반 (supanova 디자인 철학 반영)
- 모바일 퍼스트 + 데스크탑 풀폭 지원 (480px 제한 해제)
- 섹션별 시각적 차별화 (반복적 카드 행 지양)
- Framer Motion으로 스크롤 기반 애니메이션
- 최대 1개 액센트 컬러

## 레이아웃 분기

- 랜딩페이지 표시 시 기존 BottomNav, Header 숨김
- 풀스크린 레이아웃 활용
- 로그인 상태에서는 기존 480px 모바일 레이아웃 유지
